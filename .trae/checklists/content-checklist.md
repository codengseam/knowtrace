# 内容质检 Checklist

本 checklist 用于对生成的 Markdown 专栏内容进行质检与打分。配合 `.trae/skills/deep-reading/content-quality.md` 使用。

**使用方式**：逐项检查，标注 ✅ 通过 / ❌ 未通过 / ➖ 不适用。未通过项必须修复或说明原因。

---

## 一、真实性（40 分）

### 1.1 史实核验（20 分）

- [ ] 人名、时间、地点、因果与正史一致
- [ ] 关键年份已给出（改革/为相/死亡等节点）
- [ ] 典故出处准确（不张冠李戴）
- [ ] 跨文化映照事例已核验（非演义虚构）
- [ ] 史料层累/存疑说法已说明
- [ ] 无无据的人物出身或背景设定

### 1.2 名家点评（20 分）

- [ ] 只引真正对该事件/人物有明确点评的名家
- [ ] 除司马光外，至少 2 位非司马光名家
- [ ] 引文忠于原意，单段古文 ≤ 20 字
- [ ] 无「未找到」「没有明确评语」等占位
- [ ] 大意转述标注「大意据《XXX》」

## 二、可读性（30 分）

### 2.1 故事感（12 分）

- [ ] 叙事有场景、对话、戏剧性转折
- [ ] 用白话小说笔法，不堆古文再翻译
- [ ] 人物写人性矛盾和复杂面，不贴标签

### 2.2 重复控制（10 分）

- [ ] 单章节内同一事实/金句/古文只出现一次
- [ ] 章节间无大段重复讲述同一事件
- [ ] 讲人物引用前文事件时一笔带过，不展开

### 2.3 语言风格（8 分）

- [ ] 无 AI 套路句式（我们可以看到/这告诉我们/综上所述等）
- [ ] 无现代学科术语硬套历史（博弈论/底层逻辑/坐标系等）
- [ ] 无中英混杂（专有名词除外）
- [ ] 无模板过渡句堆砌（再说一层背景/这道理在历史上反复应验等）
- [ ] 段尾升华不超过 2 处，且只在问道悟道/结语

## 三、顺序（15 分）

### 3.1 叙事顺序（8 分）

- [ ] 单章内按事件发展时间线推进
- [ ] 插叙/倒叙后回到主线，倒叙标注明确时间
- [ ] 讲事情中段无过长背景打断

### 3.2 章节排序（7 分）

- [ ] 同书下章节按故事发生时间先后排列
- [ ] 文件命名与章节编号一致
- [ ] 分组目录内部也按时间先后

## 四、引用克制（15 分）

### 4.1 内联引用清理（7 分）

- [ ] 无「（见讲故事）」「（见讲事情）」「（详见下章）」「（见上文）」「（见前文）」等跳转提示
- [ ] 提及另一章时用自然语句，不加括号提示

### 4.2 行内引用密度（5 分）

- [ ] 每千字行内「——《XX·XX》」式引用 ≤ 3 处
- [ ] 同一出处连续出现只保留一处
- [ ] 普通叙述句/白话改写不挂行内出处

### 4.3 文末来源（3 分）

- [ ] 文末有「参考来源」
- [ ] 每条来源注明书名/篇名及对应内容
- [ ] 删除行内引用后未丢失来源
