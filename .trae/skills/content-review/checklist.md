# 内容质检 Checklist（六维度 v1.2）

本 checklist 用于对生成的 Markdown 专栏内容进行质检与打分。配合 `.trae/skills/deep-reading/content-quality.md` + `.trae/skills/content-review/rules/consistency-rules.md` 使用。

**v1.2 变更**：新增**一致性维度（10 分）**，从其他维度匀出（保持总分 100）。

**使用方式**：逐项检查，标注 ✅ 通过 / ❌ 未通过 / ➖ 不适用。未通过项必须修复或说明原因。

---

## 一、真实性（33 分，原 35 分，匀出 2 分给一致性）

### 1.1 史实核验（17 分，原 20 分）

- [ ] 人名、时间、地点、因果与正史一致
- [ ] 关键年份已给出（改革/为相/死亡等节点）
- [ ] 典故出处准确（不张冠李戴）
- [ ] 跨文化映照事例已核验（非演义虚构）
- [ ] 史料层累/存疑说法已说明
- [ ] 无无据的人物出身或背景设定

### 1.2 名家点评（16 分，原 20 分）

- [ ] 只引真正对该事件/人物有明确点评的名家
- [ ] 除司马光外，至少 2 位非司马光名家
- [ ] 引文忠于原意，单段古文 ≤ 20 字
- [ ] 无「未找到」「没有明确评语」等占位
- [ ] 大意转述标注「大意据《XXX》」

## 二、可读性（23 分，原 25 分，匀出 2 分给一致性）

### 2.1 故事感（10 分，原 12 分）

- [ ] 叙事有场景、对话、戏剧性转折
- [ ] 用白话小说笔法，不堆古文再翻译
- [ ] 人物写人性矛盾和复杂面，不贴标签

### 2.2 重复控制（8 分，原 10 分）

- [ ] 单章节内同一事实/金句/古文只出现一次
- [ ] 章节间无大段重复讲述同一事件
- [ ] 讲人物引用前文事件时一笔带过，不展开

### 2.3 语言风格（5 分，原 8 分）

- [ ] 无 AI 套路句式（我们可以看到/这告诉我们/综上所述等）
- [ ] 无现代学科术语硬套历史（博弈论/底层逻辑/坐标系等）
- [ ] 无中英混杂（专有名词除外）
- [ ] 无模板过渡句堆砌（再说一层背景/这道理在历史上反复应验等）
- [ ] 段尾升华不超过 2 处，且只在问道悟道/结语

## 三、顺序（13 分，原 15 分，匀出 2 分给一致性）

### 3.1 叙事顺序（7 分，原 8 分）

- [ ] 单章内按事件发展时间线推进
- [ ] 插叙/倒叙后回到主线，倒叙标注明确时间
- [ ] 讲事情中段无过长背景打断

### 3.2 章节排序（6 分，原 7 分）

- [ ] 同书下章节按故事发生时间先后排列
- [ ] 文件命名与章节编号一致
- [ ] 分组目录内部也按时间先后

## 四、引用克制（8 分，原 10 分，匀出 2 分给一致性）

### 4.1 内联引用清理（4 分，原 7 分）

- [ ] 无「（见讲故事）」「（见讲事情）」「（详见下章）」「（见上文）」「（见前文）」等跳转提示
- [ ] 提及另一章时用自然语句，不加括号提示

### 4.2 行内引用密度（3 分，原 5 分）

- [ ] 每千字行内「——《XX·XX》」式引用 ≤ 3 处
- [ ] 同一出处连续出现只保留一处
- [ ] 普通叙述句/白话改写不挂行内出处

### 4.3 文末来源（1 分，原 3 分）

- [ ] 文末有「参考来源」
- [ ] 每条来源注明书名/篇名及对应内容
- [ ] 删除行内引用后未丢失来源

## 五、灵魂（13 分，原 15 分，匀出 2 分给一致性）

### 5.1 标题灵魂点睛（5 分）

- [ ] 二级标题用描述性语言，非纯事件标签
- [ ] 标题信息密度 ≥ 4，灵魂指向 ≥ 4（详见 chapter-title-soul Skill）

### 5.2 问道悟道（5 分）

- [ ] 关键章节有「问道」段，引出人物核心抉择
- [ ] 「悟道」段不过度升华，不空泛

### 5.3 节奏呼应（3 分）

- [ ] 章节内有起承转合，非平铺直叙
- [ ] 关键金句在合适位置出现，不堆在结尾

## 六、一致性（10 分，新增）

详见 `.trae/skills/content-review/rules/consistency-rules.md`。

### 6.1 数值交叉矛盾（4 分，P0 矛盾每项扣 5 分，上限 10 分）

- [ ] 无年龄-年份数学矛盾（生年 + 事件年 ≠ 年龄）
- [ ] 无在位年数矛盾（继位年 - 去世年 ≠ 在位年数）
- [ ] 无损失-剩余数学矛盾（总数 - 损失 ≠ 剩余）
- [ ] 无百分比矛盾（部分 ÷ 总数 ≠ 百分比）

**检测命令**：
```bash
python scripts/check_consistency.py --file <文件> --archetype narrative
```

### 6.2 同事件异值（3 分，P0/P1 矛盾每项扣 3-5 分）

- [ ] 同一引文在文章内多次出现，字数标注一致
- [ ] 同一战役两次出现，兵力数字一致
- [ ] 同一典故两次出现，出处一致
- [ ] 同一人物两次出现，身份/职官一致（同一时间点）

**误报豁免**：攻防同句（A 率 N1 万破 B 的 N2 万，主动/被动语态不可比）不报；虚数前缀（数/几）与虚数后缀（余/来/多）不参与精确比较；兵力提取限定同句窗口避免跨句误抓邻战兵力（完整豁免机制见 [rules/consistency-rules.md](rules/consistency-rules.md) §2.5）。

### 6.3 实体别名冲突（2 分，P0/P1 矛盾每项扣 3-5 分）

- [ ] 同一人物字号标注一致（前文"曹操字孟德"，后文不应"曹操字子建"）
- [ ] 同一人物谥号一致
- [ ] 同一人物籍贯一致
- [ ] 同一事件朝代归属一致

**别名白名单**（合法指代，不算矛盾）：
- 曹操 ↔ 孟德 ↔ 曹孟德 ↔ 魏武帝
- 诸葛亮 ↔ 孔明 ↔ 卧龙 ↔ 丞相 ↔ 武侯
- 司马光 ↔ 君实 ↔ 温公
- 完整清单见 `src/utils/consistency.py:ENTITY_ALIASES`

### 6.4 时间线倒置（1 分，P2 矛盾每项扣 2 分）

- [ ] 年份按时间先后排列，无逆序
- [ ] 逆序段有明确倒叙标注（完整清单见 `src/utils/consistency.py:_FLASHBACK_MARKERS`，含早在/早就/要讲清/得先讲清/已经X年了/话说回/此前/在此之前/回过头看/三年前 等）
- [ ] ~~因果倒置有"起因是""缘起"等因果词~~（v1.3 路标，当前未实现）

**误报豁免**：含倒叙标注词的段落不报；仅有年份不构成事件序列的不报；"一个"量词前缀不算年号；"X年到Y年"时间范围起点不作独立时间点（完整豁免机制见 [rules/consistency-rules.md](rules/consistency-rules.md) §2.5）。
