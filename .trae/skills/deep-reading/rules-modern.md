# 现代专栏写作规则（modern 桶）

本规则用于指导 Agent 生成 modern 桶 Markdown 讲书笔记。详细项目背景见 [README.md](../../../README.md)。

> **适用范围声明**：本规则仅适用 **modern 桶**（现代方法类专栏：理财、职场、养生，如理财课、职场沟通课、面试现场、饮食/锻炼/睡眠养生课、大厂晋升指南等）。
> - **narrative 桶**（古籍专栏：史/经/子/集）见 [rules.md](rules.md)。
> - **knowledge 桶**（AI课/易经课/技术类知识专栏）见 [rules-knowledge.md](rules-knowledge.md)。
> - archetype 信源优先级与路由逻辑见 [design.md §5.6](../../../docs/archetype-design/design.md#五数据流设计)。
> - **分工边界**：本文件管「怎么写」（写作风格/结构/引用格式/叙事方法）；「怎么查」（白名单清单/阈值/扣分规则）见 [content-quality.md §8.2](content-quality.md#82-modern-桶理财职场养生) modern 桶规则集。两者协同：本文件给骨架与笔法，§8.2 给质检标尺。

## 一、结构模板（五段）

按 `config.yaml` 的 `section_templates.modern` 配置，顺序固定：

| 段名 | 作用 | 写作要点 |
|---|---|---|
| 入戏 | 场景代入，引出问题 | 用真实/可考的职场、家庭、消费场景开场，3 句话内勾住读者；避免泛泛"如今社会..."式开头 |
| 破题 | 核心观点与拆解 | 一句话点明本篇要解决什么、给出什么洞察；可借古今名家的话定调 |
| 方法论 | 可操作的方法与步骤 | 拆解为 2-4 个可执行步骤，每步配简短案例佐证；忌空谈概念 |
| 避坑 | 常见误区与反面案例 | 列 2-3 个典型踩坑场景，用真实或可考的反面案例说明 |
| 践行 | 落地建议与行动清单 | 给出 3-5 条本周可做的具体动作，量化、可验证 |

> 章节分组（如理财课「兵器与阵法/心法与避坑/守正与筑基/觉醒与破局/践行与自由」）是 chapter 维度的大类，单篇内部结构收敛为上述 5 段。

## 二、语言风格

**要做：**
- 现代白话为主，自然口语化，像樊登/脱不花讲书有节奏、有温度。
- 场景化叙事：用"会议室、地铁、家庭账本"等读者能代入的画面，而非抽象概念堆砌。
- 现代术语在描述对应行业时可用（见 §四「现代术语边界」），但首次出现建议给一句白话解释。
- 语气随段落变化：入戏生动、破题冷静、方法论清晰、避坑犀利、践行坚定。

**不做：**
- 不照搬 narrative 桶的"白话小说笔法"——现代专栏不是讲故事为主，是讲方法为主。
- 不堆砌"底层逻辑/博弈论/坐标系/底层操作系统"等大词（见 §四）。
- 不每段结尾来一句"综上所述""这告诉我们"。
- 不滥用「不是 X 而是 Y」「他不是 X，是 Y」句式（每篇 ≤ 3 处，详见 §四）。

## 三、引用规则

### 3.1 名家点评（modern 桶放宽，非古籍标准）

- **不强制司马光/臣光曰/司马迁**——现代专栏的"名家"是与该场景相关的现代或古代方法论者。
- **古今名家库**（按主题选，至少 2 位）：
  - 职场沟通：鬼谷子、韩非子、卡内基、柯维、脱不花
  - 理财：格雷厄姆、达利欧、塔勒布、芒格、巴菲特
  - 管理：德鲁克、稻盛和夫、汤姆·彼得斯
  - 心理/决策：卡尼曼、西奥迪尼、艾瑞里
  - 养生：可引《黄帝内经》等中医典籍，但须交代成书年代与争议
- **古文引用**：古代名家原文仍须真实有据、单段 ≤ 20 字、出处准确；记不清用「大意据《XXX》」转述，不可把转述伪装成原文加引号。
- **史料层累**：引用鬼谷子、战国策等有作者ship争议的典籍，必须交代成书年代/作者争议；引用苏秦等有出土文献修订的人物，必须交代（如马王堆帛书《战国纵横家书》）。
- **现代案例**：商业案例、职场经历须可考，不编造具体公司/人物/数据。

### 3.2 行内引用密度

- 关键事实、名家原话可在行内保留简短引用（「——卡尼曼《思考，快与慢》」）。
- 每千字行内引用 ≤ 3 处，同一出处连续出现只保留一处。
- 普通叙述句不挂行内出处。

### 3.3 文末来源

- 文末统一列出「参考来源」，每条注明书名/篇名及对应内容。
- 大意转述须标注「大意据《XXX》」。

## 四、现代术语边界（与 narrative 桶的关键差异）

### 4.1 大词限用

「底层逻辑」「博弈论」「坐标系」「底层操作系统」等在**描述古代历史**时禁用；在**现代职场语境**中属日常词，但仍建议替换为更朴素的表达：
- 「底层逻辑」→「根本」「本质」「关键」
- 「底层操作系统」→「底子」「根基」

全篇出现 ≤ 1 次可容忍，超过由质检标记（详见 content-quality.md §8.2.2）。

### 4.2 行业通用词白名单

行业通用词（如「bug」「KPI」等）在描述对应行业时属日常词，不算中英文混杂。完整清单（22 词）见 `src/utils/content_quality.py` 的 `MODERN_ENGLISH_WHITELIST` 常量，本文件不重复列举。

非白名单的中英紧邻混杂（如「review代码」「人品操作系统有bug」）应中文化。

### 4.3 「不是X，是Y」句式

每篇 ≤ 3 处，多余的改写为多样表达：
- 「与其说 X，不如说 Y」
- 「关键不在 X，而在 Y」
- 「X 是表象，Y 是内核」

现代职场专栏不再对单处「不是 X 而是 Y」直接报 AI 味（由本条控量，避免误报常见中文判断句）。

### 4.4 AI 味敏感模式放宽

「容易被忽略」「可见」「第[一二三四五六]层」「最关键的.*是」「这说明」「这事说明」等过于敏感的模式在 modern 桶不计为 AI 味（这些是常见中文表达）。完整清单见 `src/utils/content_quality.py` 的 `MODERN_AI_OVERSTRICT_PATTERNS`。

### 4.5 引用标注冗余

禁止「XX 在《YY》里/中讲过……（大意据《YY》）」同句重复标注。正文已写明出处的（含「在《XX》里」「在《XX》中」两种句式），句末不再挂「大意据《YY》」。

## 五、跳过项（与 narrative 桶的差异）

modern 桶**跳过**以下 narrative 桶专属检查（详见 content-quality.md §8.0 路由表）：

- `check_years_present`（年份必填）——现代专栏不强求年份
- `check_famous_critics`（古籍名家）——见 §3.1 放宽规则
- `check_temporal_order`（时间线）——现代专栏按方法论展开，不强求时间顺序
- `check_modern_jargon`（现代术语禁用）——见 §4.2 白名单
- `check_chapter_title_soul`（古籍向章回体灵魂标题）——modern 桶标题聚焦"洞察/方法/避坑"而非"事件冲突"

## 六、通用项保留（全桶共享，详见 content-quality.md）

以下检查全桶都跑，不跳过（BUG-026 教训：灵魂再好数字错了仍是 P0）：

- `check_ai_cliches`（套话黑名单）——详见 content-quality.md §9.2
- `check_numeric_facts` auto（数字硬错误）——详见 content-quality.md §9.3
- `check_numeric_facts` manual（N年/岁/品）——modern 桶过滤误标项（现代语境「10 年前」「30 岁」是正常表达）
- `check_soft_ai_pattern`、`check_redundant_citation`、`check_inline_references`/`check_sources_section`、`check_modern_jargon_terms`（硬套术语）

## 七、错别字清单与并行质检核对

详见 content-quality.md §8.5（通用错别字清单）和 §8.6（并行质检后核对五步）。modern 桶不豁免。

## 八、与文风注入的关系

modern 桶当前**跳过** tone_setter / chief_editor 节点（走原 else 分支 `orchestrator→specialists` + `quality→save`，save 链路完整不断链）。阶段4 落地 modern 版 tone_setter/chief_editor prompt 后再开启对应桶（详见 design.md §10.6）。

写作时无需考虑 soul injection——modern 桶的"灵魂"由本规则的"洞察独家性 + 落地可行性"承载，而非古籍桶的"残酷底色 + 史观穿透"。
