# 云脉·智诊伴学

> 夸克解题，钉钉批改，**云脉智诊**——做乡村师生身边的 AI 认知教研员。

## 一、项目定位

**赛道**：阿里巴巴少年云助学计划 · 乡村课堂 AI 助教（ATH 事业群 + 阿里云百炼）

**一句话定位**：不造轮子，不抢入口。做阿里教育生态（夸克搜题 - 钉钉批改）中缺失的那一层 **"认知诊断与补漏中枢"**。

- **对学生**：从"改一道错题"升级到"补一片知识"——不只给答案，更告诉你"这道题错了，本质是你 3 个年级前的 XX 知识点没掌握"。
- **对老师**：从"凭经验讲课"升级到"看数据讲课"——班级错题一键汇总、薄弱点 TOP5 置顶、**一键生成可打印的 A4 专项试卷**。
- **对乡村场景**：双轨制——学校无影云电脑走云端 Qwen3-Max，家里微信小程序走本地缓存 + 规则引擎，弱网也能用。

**MVP 范围**：初中数学七年级上册，约 50 个核心知识点，覆盖人教版四章（有理数 / 整式加减 / 一元一次方程 / 几何图形初步）。

## 二、核心竞争力

| 维度 | 为什么评委喜欢 |
|---|---|
| **生态协同** | 站在夸克 + 少年云 + 钉钉基础上做增量，不是重复造轮子 |
| **技术深度** | 轻量知识图谱 + LLM 动态推理 + RAG 防幻觉，不是套大模型壳 |
| **落地性强** | 直接对接少年云 300+ 间 AI 云教室试点，赛后即落地 |
| **社会价值** | 精准命中乡村教育"知识断层"结构性痛点 |
| **公益温度** | FunASR 方言语音周报，让不识字的祖辈也能听懂孩子学情 |

## 三、技术架构一句话

**混合架构**：轻量 JSON 知识图谱（50 节点 + 前置依赖链）+ Qwen3-Max 动态推理 + 阿里云百炼 RAG 真题检索（**绝不凭空生成题目**）+ python-docx 一键出卷 + FunASR 语音周报。

详细架构见 [docs/01-双轨制产品架构.md](docs/01-双轨制产品架构.md)。

## 五、快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置百炼平台 API Key
cp .env.example .env   # 填入 DASHSCOPE_API_KEY / QWEN_API_KEY

# 3. 校验知识图谱基座可解析
python -c "import json; d=json.load(open('data/knowledge_graph/七年级上册.json')); print(f'节点 {len(d[\"nodes\"])} / 边 {len(d[\"edges\"])}')"

# 4. 校验专栏结构（复用 HaloRead 工程化资产）
python scripts/check_book_structure.py --output content --strict

# 5. 跑测试集
pytest -q
```

## 七、核心原则

- **不造轮子**：能复用阿里生态（夸克/钉钉/百炼/无影云）就复用，做增量不做替代。
- **不抢入口**：拍照录题可对接夸克，学情推送可对接钉钉，但 MVP 阶段独立可跑。
- **防幻觉**：题目必须来自 RAG 检索真题库，LLM 只做解析、排版、变式思路，绝不凭空生成。
- **弱网可用**：核心讲解与例题静态缓存，拍照支持离线批量上传。
- **纸质友好**：所有练习题都能直接打印，不强制数字化（乡村很多学校仍是纸质作业）。
- **沉淀即闭环**：每次开发产出可校验的教训（loop_log + bug_regression_list + checklist）。

## 八、致乡村师生

> 留守在乡村的孩子放学回家，卡在一道题上，爸妈外出务工，奶奶不识字，屋里没有人能陪他解决。他把作业本一合，只能留到明天去问老师——而第二天的老师，将要面对的可能是一整间教室的孩子们。

云脉要做的，是把"诊断"这层能力补上：让错题不止于"这道怎么做"，而是"你到底哪里没学会"；让老师不止于"满堂灌"，而是"看数据讲课"。技术上我们 hold 得住，比赛逻辑上也站得住脚。
