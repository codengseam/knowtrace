# 演示剧本与提交 Checklist

> 本文档面向「云脉·智诊伴学」项目的最终提交环节，包含 3 分钟演示视频分镜表、评委视角提交前 Checklist、答辩 Q&A 预案、以及单人参赛者的测试开发差异化竞争力。所有内容以"评委看完觉得这队靠谱"为唯一目标。

### 分镜设计说明

- **三段高光对应三个用户角色**：学生（诊）→ 教师（教）→ 家长（联），闭环完整不缺角
- **每段高光都有"技术可见性"**：Omni 识别置信度、RAG 命中列表、Word 预览、FunASR 波形——评委能"看见"百炼在干活
- **公益温度在第三段集中爆发**：爷爷摸孙子头那个画面是情绪高点，前面所有技术铺垫在这里"落到人身上"
- **价值升华在最后 15 秒**：300 间教室地图 + 阿里生态拼图，把"小 demo"升维成"大落地"

## 二、评委视角提交前 Checklist

提交前逐项打勾，任何一项没做到都要补。这不是"建议"，是"硬门槛"。

### 2.1 生态对齐

- [ ] PPT/文档明确说明与夸克"互补"而非"竞争"——夸克管录题入口，云脉管诊断大脑
- [ ] PPT/文档明确说明与钉钉"互补"而非"依赖"——钉钉管触达，云脉管内容供给
- [ ] 架构图里有 `ProblemInputAdapter` / `NotificationAdapter` 抽象接口，证明"可对接"而非"硬绑定"
- [ ] PPT 单独一页讲"接入夸克生态后的放大效应"，预留 15 秒话术

### 2.2 算力现实

- [ ] 文档区分无影云（强网，云端重算力）和小程序（弱网，本地缓存兜底）两套部署形态
- [ ] 离线兜底方案写清楚：断网时无影侧缓存最近一周题库与讲义，联网增量同步
- [ ] 演示视频里至少有一个画面体现"弱网也能用"（如本地缓存命中提示）

### 2.3 防幻觉

- [ ] PPT/文档明确写出红线："出题靠 RAG 检索真题，绝不靠 LLM 凭空生成"
- [ ] 节点3（RAG）的代码层面有硬约束：下发到学生的题必须来自 RAG 命中结果，LLM 只做排序和讲义生成
- [ ] 演示视频 1:30-1:45 段有字幕叠加"题源：百炼 RAG 真题库 · 防幻觉硬约束"
- [ ] Omni 识题置信度 < 0.7 时触发"人工二次确认"分支，不强行往下走

### 2.4 公益温度

- [ ] FunASR 语音周报作为独立功能模块，不只提一句
- [ ] 演示视频第三段（家长端）是情绪高点，爷爷听语音的画面不少于 15 秒
- [ ] 语音模板设计有讲究：先进步后不足、给具体可执行动作、称呼关系而非"监护人"
- [ ] 方言覆盖写清楚 MVP 范围：普通话 + 西南官话 + 中原官话

### 2.5 百炼浓度

- [ ] 阿里生态能力映射表里至少 6 项能力（Qwen3-Omni / Qwen3-Max / 百炼RAG / 百炼Agent / FunASR / 无影云）全部有具体应用
- [ ] 演示视频每个高光段都能对应到一个阿里能力：拍题→Omni，诊断→Max+图谱，找题→RAG，出卷→Max+docx，语音→FunASR，落地→无影
- [ ] 文档里有百炼 RAG 应用搭建完整步骤（数据导入/Embedding 选型/向量库/检索调优）
- [ ] 文档里有百炼 Agent 工作流图（mermaid 或 ASCII），五个节点清楚标注用哪个阿里能力

### 2.6 落地性

- [ ] 文档写明无影云电脑预装部署方案，学校"一根网线一块屏即用"
- [ ] 文档写明与少年云 300+ 间 AI 云教室的对接思路（镜像层/账号层/数据层/试点路径）
- [ ] 演示视频 2:30-2:45 段有 300 间教室地图点亮画面
- [ ] 答辩 Q&A 有"商业化路径"预案，证明不是"做完就散"

## 三、答辩 Q&A 预案

评委大概率会问以下 5 个问题中的一个或几个。每个问题给出"评委潜台词"和"应答要点"，背熟话术。

### Q1：夸克 API 拿不到怎么办？

**评委潜台词**：你们是不是赌夸克开放？赌输了项目就废了？

**应答要点**：
1. 架构层面早就留了兜底——`ProblemInputAdapter` 接口抽象，夸克 API 不可用时，云脉自带 Omni 拍照录题，5000 题真题库 + 学生拍题双路题源，闭环不依赖夸克
2. MVP 阶段就是按"拿不到夸克"设计的，能跑通
3. 一旦夸克开放，是放大效应不是生存依赖——题源从 5000 扩到百万级，诊断中枢价值指数级放大
4. 收口话术："我们不赌夸克开放，但我们的架构为夸克开放准备好了。"

### Q2：知识图谱准确性如何保证？

**评委潜台词**：图谱是你们自己画的，错了怎么办？诊断不就成了错上加错？

**应答要点**：
1. 图谱来源：基于人教版/北师大版/苏教版教材目录 + 中考考纲 + 一线教研老师审校，380 节点 1200 边，不是 LLM 生成的
2. 持续校验：每学期由试点学校教研组抽审 10% 节点的 `predecessors` 关系，错标率压到 3% 以下
3. 诊断不唯图谱——Max 在诊断时会同时输出"图谱路径"和"题面错因分析"双信号，老师可见可纠
4. 反馈闭环：教师端有"诊断不准"反馈按钮，错例回流到图谱迭代队列
5. 收口话术："图谱是地基不是天花板，有教研审校 + 教师反馈双闭环持续修正。"

### Q3：乡村老师会用吗？

**评委潜台词**：你们这帮人做的东西太复杂，乡村老教师学不会。

**应答要点**：
1. 教师端就三个按钮：看学情、一键出卷、导出周报。没有"配置""调参""选模型"
2. 出卷流程：输入薄弱点（系统已诊断好）→ 点生成 → 3 秒出 A4 Word → 打印。全程不碰 AI 概念
3. 落地形态是少年云无影云桌面预装，老师开机就见，不用装软件、不用注册
4. 试点阶段配 1 对 1 教研陪跑，第一个月每周一次线上答疑
5. 收口话术："我们没让老师学 AI，我们让 AI 躲在老师熟悉的'出卷子'动作后面。"

### Q4：大模型出数学题翻车怎么办？

**评委潜台词**：LLM 出数学题符号错、逻辑错、超纲，你们怎么防？

**应答要点**：
1. **硬约束**：所有下发到学生的题，必须来自百炼 RAG 检索的真题库命中结果，LLM 不生成题面
2. LLM 只做三件事：识别题面（Omni）、诊断推理（Max 读图谱）、生成讲义和排版（Max 填模板）
3. RAG 命中后还有 Rerank 精排 + 难度 filter，跨难度误召回率 ≤ 5%
4. 教师端讲义版每题标注"题源 problem_id"，可追溯回真题库原题，老师可核验
5. 收口话术："我们不让大模型出题，我们让大模型选题和讲题。出题这件事，交给真题库。"

### Q5：商业化路径？

**评委潜台词**：公益项目不能只讲情怀，能不能活下去？

**应答要点**：
1. 公益侧：少年云 300+ 间乡村教室免费使用，基础诊断+出卷+语音周报全免费
2. 增值侧：县域教育局学情看板、教研组协同备课、定制讲义模板——面向有预算的区县教育部门收费
3. 生态侧：对接夸克后，夸克错题本导流到云脉诊断，按调用次数分润；对接钉钉后，钉钉班级群学情卡片由云脉供给
4. 数据侧：脱敏学情数据反哺教材出版社（知识点难度校准），形成数据飞轮
5. 收口话术："乡村免费做公益，县域增值做商业化，生态对接做分润。公益不是不赚钱，是赚的是生态位的钱。"

## 四、测试开发优势降维打击

作为单人参赛者，最大劣势是"人少做不完"，最大优势是"测试开发背景天然防翻车"。这一节把测试开发能力转化为评委可见的差异化竞争力。

### 4.1 兜底与鲁棒性（try-except / 重试 / 默认推荐库）

**评委痛点**：现场演示翻车是评审大忌，多团队死在这上面。

**云脉的兜底体系**：

| 环节 | 失败场景 | 兜底动作 |
|---|---|---|
| Omni 识题 | 置信度 < 0.7 / 接口超时 | 触发"人工二次确认"分支 + 重试 3 次 + 默认推荐题库兜底 |
| Max 诊断 | 接口限流 / 返回非 JSON | 降级到"基于知识点直接检索"的弱诊断模式，不阻塞主链路 |
| RAG 检索 | 召回为空 | 兜底取"同章节最近难度"题，保证有题可出 |
| Word 排版 | python-docx 异常 | 降级到纯文本讲义 + 提示教师手动排版 |
| FunASR | 方言模型未加载 | 降级到普通话 TTS，不报错 |
| 无影云断网 | 本地缓存命中 | 提示"离线模式"，用最近一周缓存题库 |

**话术**："我一个人参赛，所以每个环节都假设它会挂。挂了不黑屏，挂了有兜底。这是测试开发的本能。"

### 4.2 边界测试用例构造

**评委痛点**：很多队伍只测"正常流程"，一遇到脏数据就崩。

**云脉的边界用例集**（演示前全部跑过）：

- **模糊图片**：手抖/光线暗/部分遮挡 → Omni 应输出低置信度并触发人工确认，不强行识别
- **乱写草稿**：学生拍的不是题是草稿纸 → Omni 识别"非题面"标签，系统提示"请重新拍题"
- **断网重连**：演示中途拔网线 5 秒再插回 → 本地缓存继续可用，重连后增量同步不丢数据
- **公式渲染崩溃**：题面含复杂 LaTeX（如分式套根号）→ Word 排版用图片化公式兜底，不依赖 Word 原生公式编辑器
- **超长题面**：应用题 300 字以上 → RAG 切片不截断，整题入库
- **知识点冷门**：检索的薄弱点是冷门节点（如"十字相乘法"）→ RAG 召回不足时，扩展到上级节点（"因式分解"）兜底
- **方言不支持**：监护人方言不在 MVP 三种内 → 自动降级普通话 + 文字提示

**话术**："演示用的'完美错题案例'是我从 200 个边界用例里挑出来的，不是碰巧跑通的。剩下 199 个 case 我都测过不会翻车。"

### 4.3 演示视频不翻车的"完美错题案例"构造

**评委痛点**：演示视频里的错题太巧了，评委怀疑是"导演的"。

**云脉的策略——用真实边界用例当演示案例**：

1. **拍题对象**：选一道真实八年级一元一次方程错题，学生手写有涂改、有红叉，不是干净打印题
2. **诊断深度**：这道题的根因要能溯源到一个"出人意料但合理"的前置薄弱点（如"移项符号"溯源到"等式性质"），让评委"哦原来如此"
3. **RAG 命中**：提前测过这道题的薄弱点，RAG Top-3 命中率 100%，演示时不会出现"没找到题"的尴尬
4. **Word 排版**：模板提前固化，Max 只填内容，3 秒生成稳定可复现
5. **语音周报**：脚本提前写好，FunASR 合成稳定，方言选择西南官话（覆盖广、效果好）

**话术**："这个案例是真实的，但真实不是随机。我从 200 个边界 case 里选了这个——它真实、它有诊断深度、它每一步都不翻车。这是测试开发教给我的：演示不是碰运气，是选 case。"

### 4.4 测试开发优势的评委话术收口

> "我一个人参赛，所以我没法靠人海战术堆功能。但我有测试开发背景，这意味着：第一，每个环节都有兜底，演示不翻车；第二，边界 case 都测过，脏数据不崩；第三，演示案例是选出来的不是碰出来的。别人比的是功能多，我比的是功能稳。乡村课堂用东西，稳比多重要。"

这段话建议在答辩最后 30 秒主动抛出，把"单人参赛"的劣势翻转成"测试开发稳如狗"的优势，评委记忆点极强。
