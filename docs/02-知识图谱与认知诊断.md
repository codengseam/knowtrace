# 02 知识图谱与认知诊断

> 本文是「云脉·智诊伴学」项目的核心算法与产品说明，回答三个问题：
> 知识怎么存？错在哪？怎么补？以及教师最关心的——能不能一键出一张卷子。

---

## 一、知识图谱设计哲学：为什么是“轻量JSON”而不是 Neo4j

一句话：**单人开发、可控可校验、可 git 版本化**。

### 1.1 我们没用图数据库，是有意为之

七年级上册就 50 个左右的知识点，节点规模撑死三位数。这种量级上 Neo4j 是用大炮打蚊子——要部署服务、配 Docker、写 Cypher 查询、维护连接、担心重启丢数据。对乡村课堂的部署环境（一台云电脑、可能断网）来说，这是反优化。

我们选的路是：

- **一张 JSON 文件 = 整个图谱**。文件名 `七年级上册.json`，约几十 KB，肉眼可读。
- **节点和边都长在同一个文件里**。`nodes` 数组放知识点，`edges` 数组放依赖关系。
- **校验靠脚本**。一条 `python -c "import json; json.load(open('...'))"` 就能保证可解析；再加一个无环检测脚本，就能保证拓扑合法性。
- **版本化天然支持**。`git diff` 一次提交，谁加了哪个知识点、谁改了哪条依赖链，清清楚楚。这对教学产品迭代尤其重要——教研老师改一版，开发同步一版，全程留痕。

### 1.2 那“动态推理”交给谁？交给 LLM

图谱负责“静态骨架”：哪些知识点存在、它们的前置是谁、学生常见错因是什么。
LLM（Qwen3 系列）负责“动态血肉”：拿到一道错题，结合图谱上下文，推断学生卡在哪条链上、需要补哪个节点。

这就是项目核心算法的「**轻量级 JSON 图谱 + LLM 动态推理**」混合模式。图谱小而稳，LLM 灵而准，各取所长。

### 1.3 为什么这套方案能在乡村课堂跑起来

- **零基础设施**：不需要图数据库服务，一份 JSON 文件跟随代码仓库走。
- **离线友好**：断网时图谱本身可用，LLM 调用走百炼 API（弱网也能跑通单次推理）。
- **教研可参与**：教研老师打开 JSON 看 `error_types` 字段，能直接补充学生错因，不需要懂代码。
- **可演进**：未来扩展到八年级、九年级，只需新增文件，不破坏既有结构。

## 二、七年级上册图谱节点统计

### 2.1 按章分布

| 章节 | 节点数 | 章节定位 |
|---|---|---|
| 第1章 有理数 | 17 | 正负数→数轴→相反数→绝对值→加减乘除乘方→科学记数法→近似数 |
| 第2章 整式的加减 | 9 | 字母表示数→单项式/多项式/整式→列代数式→同类项→去括号→整式加减 |
| 第3章 一元一次方程 | 11 | 方程概念→等式性质→解方程四步法→行程/工程/利润/分配应用 |
| 第4章 几何图形初步 | 13 | 立体平面图形→展开图/三视图→点线面体→线段→角→余补角→方位角 |
| **合计** | **50** | 覆盖人教版七上全部核心小节 |

> 第3章多算了一个综合节点 `K7A050 一元一次方程应用综合`，用于诊断学生“题型识别”这一元能力。

### 2.2 按难度分布

| 难度 | 含义 | 节点数 | 占比 |
|---|---|---|---|
| 1 | 基础（概念识记） | 15 | 30% |
| 2 | 中等（运算/规则应用） | 27 | 54% |
| 3 | 较难（综合应用/多步推理） | 8 | 16% |

难度分布刻意做成“中间大、两端小”：基础题保底、难题做区分度，匹配乡村学生水平梯度。难度2占大头（54%），因为七上主旋律是运算规则应用，不是高维推理。

### 2.3 节点 ID 规范

- 前缀 `K7A` = 七年级上册（K=知识点，7=七年级，A=上册）。
- 后跟 3 位序号，`K7A001`~`K7A050`，全局唯一。
- 序号按章节顺序分配，肉眼一看 ID 就知道大致在哪一章。

### 2.4 边统计

图谱共 68 条 `prerequisite` 边，平均每个节点入度约 1.4。节点最多前置的是 `K7A016 有理数混合运算综合`（3个前置）和 `K7A050 一元一次方程应用综合`（4个前置），这两个节点是综合应用型，前置密集符合预期。

## 三、前置依赖链示例

依赖链是认知诊断的灵魂——错了一道难题，要能溯源到最底层的概念缺漏。下面举 3 条典型链路。

### 3.1 链路一：“一元一次方程应用题”完整溯源链

学生做错一道**行程问题**应用题，可能的溯源链：

```
K7A033 行程问题应用
  ↑ 依赖
K7A032 去分母解一元一次方程
  ↑ 依赖
K7A031 去括号解一元一次方程
  ↑ 依赖
K7A030 合并同类项与移项解方程
  ↑ 依赖
K7A029 等式的性质
  ↑ 依赖
K7A028 一元一次方程概念
  ↑ 依赖
K7A026 整式加减运算
  ↑ 依赖
K7A024 合并同类项 + K7A025 去括号
  ↑ 依赖
K7A023 同类项概念 + K7A021 整式
  ↑ 依赖
K7A019 单项式
  ↑ 依赖
K7A018 用字母表示数
  ↑ 依赖
K7A016 有理数混合运算综合
```

**诊断含义**：学生卡在行程题，表面看是“题没读懂”，沿链向上溯源，真正薄弱点可能是 `K7A025 去括号`（括号前负号没全变号），导致解方程时整道崩盘。补漏路径优先补去括号，而不是反复刷行程题。

### 3.2 链路二：“角的运算”跨章溯源

学生做错**角的和差计算**题，溯源链：

```
K7A046 角的比较与运算
  ↑ 依赖
K7A045 角的度量（度分秒）
  ↑ 依赖
K7A044 角的概念与表示
  ↑ 依赖
K7A041 直线、射线、线段（角的边来自射线）
  ↑ 依赖
K7A040 点、线、面、体
  ↑ 依赖
K7A037 立体图形与平面图形

同时（跨章依赖！）：
K7A046 角的比较与运算
  ↑ 依赖
K7A009 有理数加减混合运算（第1章）
```

**诊断含义**：几何图形章节相对独立，但“角的运算”本质上是有理数加减的几何化应用。学生如果角度运算老算错，根因可能在第1章的符号处理，而不是几何概念没记住。这就是混合模式的价值——**LLM 跨章推理**，而图谱只是给 LLM 提供可走的路径。

### 3.3 链路三：“科学记数法”的单点深挖

```
K7A014 科学记数法
  ↑ 依赖
K7A013 有理数的乘方
  ↑ 依赖
K7A010 有理数乘法及运算律
  ↑ 依赖
K7A005 绝对值
  ↑ 依赖
K7A004 相反数
  ↑ 依赖
K7A003 数轴
  ↑ 依赖
K7A002 有理数及分类
  ↑ 依赖
K7A001 正数与负数
```

**诊断含义**：科学记数法表面是“格式题”，但学生写不对 `a×10ⁿ` 中 a 的范围，往往是因为乘方概念（`K7A013`）没建立好，再往下挖可能是乘法符号判定（`K7A010`）有漏洞。链路够深，诊断才够准。

## 四、认知诊断算法流程

诊断的输入是一道错题，输出是一份“你卡在哪、怎么补”的报告。流程分四步：

### 4.1 错题归集

学生做完练习或测验，错题自动进入诊断队列。每道错题至少携带：

- 题干、学生作答、正确答案
- 题目标注的知识点 ID（出题时就绑定，比如这道题考 `K7A032`）
- 学生作答时间戳（用于判断“是真不会”还是“时间不够”）

### 4.2 Qwen3-Omni 多模态识别

错题可能是一张拍照（学生手写作业），用 **Qwen3-Omni** 做第一步处理：

- 图像→文本：把手写算式 OCR 成结构化文本。
- 识别学生在哪一步出错（比如“去分母时漏乘了第二项”）。
- 输出结构化的错因标签，喂给下一步。

> 用 Qwen3-Omni 的理由：乡村学生交作业多为拍照上传，多模态识别是刚需，不是花架子。

### 4.3 知识点定位

拿到错因后，结合题目预标注的知识点 ID，初步锁定责任节点。但仅靠题目标注不够——一道应用题可能同时考“去括号”和“行程情境”，需要 LLM 判断主要责任在哪个节点。

这一步用**轻量图谱检索**：从题目知识点出发，沿 `prerequisites` 反向走 1~2 层，列出候选责任节点集合。

### 4.4 Qwen3-Max 推理前置薄弱点

把候选节点集合 + 学生历史错因 + 图谱中每个节点的 `error_types` 字段，一起喂给 **Qwen3-Max**，让它做深度推理：

- 对比学生这次的错因和每个候选节点的典型错因，匹配度高的就是真薄弱点。
- 如果根因在前置节点（比如解方程错，根因在去括号），沿依赖链向上溯，找到最深的“红色”节点。
- 输出一份补漏路径：**先补哪个节点 → 再补哪个节点 → 最后回到当前题练习**。

### 4.5 生成补漏路径与练习推送

补漏路径生成后，系统自动给学生推送：

1. **概念微课**（30秒~2分钟视频或图文卡片）针对根因节点。
2. **一道变式题**（同知识点、同难度、不同情境）检验是否补上。
3. 通过后再回到原错题重做，形成闭环。

```
错题 → Qwen3-Omni识别 → 知识点定位 → Qwen3-Max推理前置薄弱点 → 补漏路径 → 推送微课+变式 → 闭环重做
```

整条链路对学生透明，学生只看到“这里有个小视频，看完做道题”。对教师透明度更高——后台能看到每个学生的薄弱节点热力图。

## 五、教师端“一键出卷”流程

这是乡村教师呼声最高的功能：**别让我手抄题、别让我排版、给我一张能直接印能直接考的卷子**。

### 5.1 输入

教师在 Web 界面填三样东西：

| 输入项 | 取值 | 说明 |
|---|---|---|
| 薄弱知识点 | 多选节点 ID | 来源：班级诊断报告，教师勾选“最近全班都在错”的 2~5 个知识点 |
| 难度档位 | 基础 / 提升 / 拓展 | 基础=difficulty 1+2；提升=2+3；拓展=全 3 |
| 题量 | 整数（默认 10） | 5~30 之间，自动按知识点均匀分配 |

教师还可以选“是否包含应用题”（应用题篇幅大，影响题量上限）。

### 5.2 流程

```
教师输入(薄弱点+难度+题量)
    ↓
百炼 RAG 检索真题
    （从沉淀的题库向量检索，按知识点+难度召回候选题）
    ↓
Qwen3-Max 排版与解析生成
    （去重、按难度梯度排序、为每题生成规范解析步骤）
    ↓
python-docx 生成 A4 Word 文档
    （卷头、题号、装订线、答案分页、字号字体统一）
    ↓
可选：转 PDF（用 libreoffice --headless --convert-to pdf）
    ↓
教师下载，直接打印
```

**关键技术点**：

- **百炼 RAG 检索**：题目存在向量库里，按知识点 ID + 难度做语义检索，召回 Top-N 候选题。题目来源是教研团队沉淀的真题库（含人教版教材原题、各地期末真题、变式题）。
- **Qwen3-Max 排版**：题目检索回来可能重复、可能难度不均，让 Qwen3-Max 做二次筛选和排序，保证“由易到难、知识点覆盖均衡”。同时为每道题生成**规范解析**（分步、标注依据），供教师参考。
- **python-docx 出卷**：用 python-docx 程序化生成 Word，而不是让 LLM 直接吐 Word——LLM 排版不稳定，python-docx 能精确控制字号、行距、装订线、分栏。这一步是“确定性”的，确保每张卷子长得一样规整。
- **可选 PDF**：乡村学校打印机如果只接 PDF，用 `libreoffice --headless --convert-to pdf` 把 Word 转成 PDF。这一步在服务器侧完成，教师不需要装任何软件。

### 5.3 输出样例描述

一张标准 A4 试卷的样貌：

- **卷头**（约占顶部 1/5 页面）：
  - 校名 + 年级 + 「数学随堂练习」标题
  - 班级、姓名、学号三栏（手填用）
  - 考试时长、满分、出卷日期
  - 一行装订线提示（左侧留 2cm 装订区）

- **正文题区**：
  - 一、选择题（题号 1~N，每题 4 个选项，留答题框）
  - 二、填空题（题号 N+1~M，留横线）
  - 三、解答题（题号 M+1~，留作答空间，应用题空间加大）
  - 题号统一用“一、1.”格式，字号小四，行距 1.5 倍
  - 难度按“基础→提升→拓展”梯度排列

- **答案与解析分页**：
  - 答案单独起一页，方便教师拆开印
  - 每题答案 + 解析步骤（标注知识点 ID，便于教师讲评时定位）
  - 解析用五号字、灰色，与正文区分

- **页脚**：页码、出卷时间戳、知识点 ID 清单（教师扫一眼就知道这卷考了什么）

教师拿到 Word 后，可选两种用法：

1. **直接打印发卷**：闭卷考。
2. **拆出答案页**：答案页单独印给教师讲评用，学生卷不留答案。

## 六、学生端“认知诊断树”可视化设计

学生在自己的页面看到的是一棵**会变色的知识树**，而不是一份冷冰冰的成绩单。

### 6.1 四色节点含义

| 颜色 | 含义 | 触发条件 |
|---|---|---|
| 🟢 绿色 | 已掌握 | 该节点最近 3 次练习正确率 ≥ 85% |
| 🟡 黄色 | 部分掌握 | 正确率 60%~85%，或最近一次出错但前置节点是绿色 |
| 🔴 红色 | 薄弱 | 正确率 < 60%，或前置节点存在红色（连锁薄弱） |
| ⚪ 灰色 | 未练习 | 该节点从未做过题，无数据 |

**关键设计：红色会“传染”**。如果一个节点本身没做过题，但它的前置节点是红色，这个节点也显示成浅红——提示学生“这个还没学，但你的基础没打牢，学之前先补基础”。

### 6.2 点击节点展开

学生点击任意节点，弹出三层卡片，按需展开：

1. **概念层**：一句话讲清这个知识点是什么 + 30 秒微课视频。
2. **例题层**：1 道带详细解析的典型例题，分步标注。
3. **变式层**：1~3 道变式题，难度递增，做完即时反馈对错，对错回写诊断树更新颜色。

设计原则是**渐进披露**：学生不需要一次看 5 道题，先看概念，再看例题，再动手做变式。这符合乡村学生的认知节奏——一次喂太多会劝退。

### 6.3 树的可视化形态

不做传统树状图（七上 50 个节点画成树会很挤），改用**章节分组 + 进度条**：

- 顶部 4 个章节卡片，每个卡片显示该章的绿色/黄色/红色/灰色节点数。
- 点击章节卡片，展开该章的节点列表，每个节点是一个小圆点（带颜色）。
- 节点圆点可点击，展开三层卡片。
- 整页底部有一个“补漏建议”横幅，列出当前最该补的 3 个红色节点（按依赖深度排序，先补最底层）。

这种设计在手机上也能看清，乡村学生很多是手机端使用。

## 七、评委视角加分点

如果评委问“这方案凭什么能赢”，我们有 5 条硬理由：

### 7.1 混合模式兼顾“稳”与“准”

纯图谱方案（Neo4j）部署重、维护贵、乡村跑不动；纯 LLM 方案无结构、不可校验、容易“一本正经胡说八道”。我们的「轻量 JSON + LLM 动态推理」把静态骨架和动态血肉分开——骨架小而稳（git 可版本化、脚本可校验），血肉灵而准（LLM 跨章推理、按需调用）。**这是单人开发能跑通、乡村课堂能落地的唯一解**。

### 7.2 跨章溯源能力

很多诊断系统只看“这道题考哪个知识点”，我们能做到跨章溯源——几何章节的“角的运算”能溯源到第1章“有理数加减”。这背后是图谱里**跨章依赖边**的设计（如 `K7A009 → K7A046`），加上 LLM 沿边推理。这意味着补漏不只是“补这一节”，而是“补到根”，避免“反复刷同类型题但永远学不会”。

### 7.3 教师端真·一键出卷

不喊“AI 出卷”口号糊弄，而是把流程拆成**可确定性保证的环节**：

- 题目检索靠 RAG（可溯源、可去重）。
- 排版靠 python-docx（字号字体装订线精确控制）。
- 解析靠 Qwen3-Max（分步、标注知识点 ID）。
- PDF 转换靠 libreoffice（无需教师装软件）。

每一步都可解释、可校验、可替换。教师拿到的是**能直接印的卷子**，不是“AI 生成的文本还要自己排版”。这对乡村教师是真正的减负——他们没有时间也没有工具去手工排版。

### 7.4 多模态识别贴合乡村真实场景

乡村学生交作业的主流方式是**拍照上传**，不是 PC 端打字。我们用 Qwen3-Omni 做手写识别 + 错因定位，不是炫技，是对真实场景的尊重。这意味着学生不需要改变交作业的习惯，老师不需要重新培训——这套系统“长”在现有流程上，而不是要求流程来适应系统。

### 7.5 可演进、可复制

- 七上图谱跑通后，**八上、九上只是新增 JSON 文件**，不破坏既有结构。
- 图谱 schema 是公开的，教研团队可以自己维护、自己增删节点，不依赖开发。
- 一键出卷的题目来源是可累积的题库，越用越准（RAG 检索质量随题库增长提升）。

这意味着这套方案在评奖之后还能继续跑——不是一次性 demo，而是一个**能迭代、能复用、能扩散到其他学科**的底座。这正是“少年云助学计划”想要的——给乡村课堂一个能自己长起来的工具，而不是一个靠输血维持的系统。
