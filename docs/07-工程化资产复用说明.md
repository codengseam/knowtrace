# 07 · 工程化资产复用说明

本文档记录 knowtrace 项目从 HaloRead（姊妹项目）复用的工程化资产清单、调整说明与使用方式。

## 一、为什么复用

作为单人参赛者，时间是最稀缺资源。HaloRead 项目已沉淀一套成熟的工程化体系（12 Skill + 校验脚本 + CI + bug 回归 + loop_log），knowtrace 直接复用可省去 2-3 周的基础设施搭建时间，把精力聚焦在比赛核心链路（诊断 / RAG / 出卷）。

## 四、使用方式

### 4.1 日常开发

直接用 Trae IDE 打开 knowtrace 项目，`.trae/rules/` 会自动加载，五步协作流程自动生效。用户无需手动粘贴提示词。

### 4.2 合并前校验

```bash
# 三道门禁必须全过
python scripts/check_book_structure.py --output content --strict
pytest -q
bash tests/run_regression_suite.sh
```

### 4.3 提交信息规范

```bash
python scripts/validate_commit_messages.py origin/master..HEAD
```

标题与正文均须中文，且准确概括当前修改。

### 4.4 loop_log 沉淀

每次开发完成后，按 dev-workflow 第五步：

1. 只 append 到当月分片 `docs/loop_log/YYYY-MM.md`
2. 运行 `python scripts/regen_loop_log_index.py` 重生成主文件索引
3. 运行 `python scripts/check_loop_log.py` 校验

## 五、不复用的部分

以下 HaloRead 资产**不迁移**到 knowtrace（避免污染比赛项目）：

- `book-sources/`：HaloRead 书源
- `output/`：HaloRead 其他专栏（史/经/养生/财/技/职场）
- `site/`：HaloRead 静态站点产物
- `demos/`：HaloRead demo
- `docs/`（除 loop_log 外）：HaloRead 专属设计文档
- `prompts/`：HaloRead 讲书提示词（knowtrace 用 `data/prompts/` 存诊断/出卷 Prompt）
- `src/agents/`：HaloRead 13 个 Agent（knowtrace 走百炼 Agent 编排，不用本地 LangGraph）
