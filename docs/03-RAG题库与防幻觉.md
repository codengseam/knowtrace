# RAG 题库与防幻觉技术方案

> 适用项目：云脉·智诊伴学（"云脉"）
> 推荐模型族：Qwen3-Max / Qwen3-Omni / Qwen3 端侧小尺寸 / FunASR
> 核心架构：轻量 JSON 知识图谱 + Qwen3-Max 动态推理 + RAG 防幻觉

---

## 1. 防幻觉核心原则

### 1.1 核心铁律

**题目必须来自 RAG 检索真题，LLM 绝不凭空生成题目。**

### 1.2 为什么必须如此

数学题对正确性零容忍。一道几何题，条件少给一个，整道题无解；数值改一个数字，答案可能从"存在"变"不存在"；选项错位，整道选择题就废了。这与写诗、写文案完全不同——后者差一个字最多不优美，数学题差一个字就是错题。

LLM 生成数学题的典型翻车场景，已在前期调研中反复验证：

| 翻车类型 | 典型表现 | 后果 |
|---|---|---|
| 条件缺失 | 三角形题目不给"两边及其夹角"，直接问面积 | 题目无解或多解 |
| 数值不自洽 | 给定边长违反三角形不等式 | 题目数学上不存在 |
| 答案与题干不符 | 题干问"最大值"，答案给的是"最小值" | 学生越练越错 |
| 知识点错挂 | 把"二次函数图象"题标成"一元一次方程" | 诊断结论全错 |
| 解析跳步 | 关键变形不写，直接"由题意得" | 学困生看不懂 |

只要允许 LLM 生成题目，上述五类问题**无法根除**——这是模型能力边界，不是 prompt 调优能解决的。唯一可靠路径：**题目从已审核真题库中检索，LLM 只做"选、排、补、变"四件事**：

- **选**：从检索结果中挑选最匹配薄弱知识点的题目；
- **排**：按难度递进、知识点覆盖重新排版成卷；
- **补**：对检索结果的解析做查漏补缺（不改题干和答案）；
- **变**：基于真题生成"变式题的解题思路"（不是变式题本身，变式题仍由真题库筛选）。

### 1.3 防幻觉三道闸门

1. **数据闸门**：题库全部来自人工审核的真题，源头即"已对"。
2. **检索闸门**：题目进入 LLM 上下文必须经过向量检索 + 元数据过滤，不进入上下文的题目不可能出现在出卷结果里。
3. **校验闸门**：LLM 输出后做 schema 校验 + 题目 ID 回查（输出题目必须在检索列表中），校验不过即降级。

## 2. 数据准备方案

### 2.1 题库来源

| 来源 | 量级 | 权威性 | 备注 |
|---|---|---|---|
| 近 5 年各地中考真题 | ~3000 道 | 最高 | 主干，覆盖核心考点 |
| 开源题库（教育部基础教育资源平台等） | ~1500 道 | 高 | 补充题型多样性 |
| 人教版初中数学教材课后题 | ~500 道 | 高 | 覆盖基础概念题 |

合计约 **5000 道初中数学题**，覆盖人教版七至九年级全部章节。

### 2.2 切片策略

每道题按以下结构化 JSON 存储，**不做段落切片**——数学题是原子单元，拆开就废了。

```json
{
  "question_id": "GD-2024-001",
  "source": "2024年广东省中考数学第15题",
  "stem": "已知二次函数 $y=ax^2+bx+c$ 的图象经过点 $(-1,0)$、$(3,0)$、$(0,-3)$，求该二次函数的解析式。",
  "answer": "$y=x^2-2x-3$",
  "analysis": "由图象与 $x$ 轴交点 $(-1,0)$、$(3,0)$ 可设交点式 $y=a(x+1)(x-3)$；代入 $(0,-3)$ 得 $a\\cdot 1\\cdot (-3)=-3$，解得 $a=1$，故 $y=(x+1)(x-3)=x^2-2x-3$。",
  "knowledge_points": ["KP-FUNC-003", "KP-FUNC-007"],
  "difficulty": 2,
  "difficulty_label": "中档",
  "question_type": "解答题",
  "grade": "九年级",
  "year": 2024,
  "region": "广东"
}
```

字段说明：
- `knowledge_points`：知识点标签数组，对应知识图谱节点 id；
- `difficulty`：1 基础 / 2 中档 / 3 拔高 / 4 竞赛拓展；
- `question_type`：选择题、填空题、解答题、证明题、应用题；
- `stem` / `answer` / `analysis`：均使用 LaTeX，行内 `$...$`，独立块 `$$...$$`。

### 2.3 向量化

- **Embedding 模型**：阿里云百炼平台 `text-embedding-v3`（中文优化，1024 维）。
- **向量化对象**：`stem + " " + analysis + " " + knowledge_points 名称拼接`——把题干、解析、知识点三者一起 embed，避免"题面相近但考点不同"的题目被误检索。
- **元数据不向量化**：`difficulty`、`question_type`、`grade` 作为元数据（metadata）单独存储，用于结构化过滤，不参与向量相似度计算。

### 2.4 向量库

- **载体**：阿里云百炼平台向量数据库（与 Embedding 模型同源，免跨平台搬运）。
- **索引类型**：HNSW（图索引，召回速度与精度平衡）。
- **partition 策略**：按 `grade` 分区（七 / 八 / 九年级），减少跨学段误检索。
- **更新机制**：题库每月批量补充新真题，全量重建索引（数学题库变更频率低，无需实时增量）。

## 3. RAG 检索流程

### 3.1 Query 构造

检索 query 不是学生原始输入，而是**诊断后组装的结构化查询**：

```
query_text = 薄弱知识点名称拼接 + " " + 题型关键词
filters = {
    "difficulty": 难度档位,         # 如 2（中档）
    "question_type": 题型,          # 如 "解答题"
    "grade": 学段,                  # 如 "九年级"
    "knowledge_points": [kp_id]     # 必含至少一个薄弱知识点
}
```

示例：诊断结果显示学生在"二次函数图象与系数关系"（KP-FUNC-007）上薄弱，难度档位 2，题型"选择题"，则：

```
query_text = "二次函数图象开口方向 顶点坐标 对称轴 a b c 系数 选择题"
filters = {difficulty: 2, question_type: "选择题", grade: "九年级", knowledge_points: ["KP-FUNC-007"]}
```

### 3.2 Top-K 检索

- **K = 10**：在 5000 题量级下，K=10 兼顾覆盖与去噪；过小则题型单一，过大则稀释相关性。
- **两阶段过滤**：
  1. 元数据硬过滤（difficulty / question_type / grade / knowledge_points 必须命中）；
  2. 向量相似度 soft ranking，取前 10。
- **去重**：同一考点不允许出现 3 道以上同型题，按 `knowledge_points` 聚合后限流。

### 3.3 Qwen3-Max 二次筛选与排版

LLM 在这一步**不生成题目**，只做四件事：

| 动作 | 输入 | 输出 | 是否允许改原文 |
|---|---|---|---|
| 选 | 10 道候选题 + 学生薄弱画像 | 选中的 N 道（N 由"题量"参数决定） | 否 |
| 排 | 选中的 N 道题 | 按难度递进 + 知识点交替重排 | 否（仅顺序） |
| 补 | 候选题中解析缺失 / 简略的题 | 补全解析（保留原解析，做加法不做减法） | 是（仅解析字段） |
| 变 | 选中的题目 | 为每道题生成"变式解题思路"（不是变式题本身） | 是（新增字段） |

**关键约束**：输出 JSON 中的 `question_id` 必须存在于检索候选列表中，校验失败即降级到"默认推荐题库"。

## 4. LLM 角色分工表

| 模型 | 职责 | 输入 | 输出 | 何时调用 |
|---|---|---|---|---|
| **Qwen3-Omni** | 多模态识别手写错题 | 学生拍照上传的错题图片 | 题干文本 + 学生错解文本（含 LaTeX 算式） | 学生上传错题时 |
| **Qwen3-Max** | 认知诊断推理 | {题目、学生错解、知识图谱 JSON 片段} | 错因诊断 JSON（含错因类型、薄弱前置知识点、辅导话术、补漏路径） | 学生提交错题后、出卷前 |
| **Qwen3-Max** | 出卷排版 | {薄弱知识点、难度档位、检索真题列表、题量} | 试卷 JSON（题目数组，仅来自检索列表） | 教师 / 学生点击"一键出卷"时 |
| **Qwen3-Max** | 教研讲义生成 | {班级错题统计、知识图谱} | 讲义 Markdown（含考点梳理、典型错例、变式训练思路） | 教师点击"生成讲义"时 |
| **Qwen3 端侧小尺寸** | 弱网降级基础识题 | 错题图片 | 题干纯文本（无 LaTeX、无错解识别） | 弱网 / 断网时，仅完成"识题入库"，诊断延后到网络恢复 |

### 调用优先级

```
正常网络 → Qwen3-Omni 识题 → Qwen3-Max 诊断 → RAG 检索 → Qwen3-Max 出卷
弱网     → Qwen3 端侧识题（仅入库）→ 网络恢复后补诊断
```

## 5. 数学公式处理方案

### 5.1 识别

- **主路径**：Qwen3-Omni 多模态识别，提取题干中核心算式，输出 LaTeX（行内 `$...$`、独立块 `$$...$$`）。
- **不强求完美 LaTeX**：手写体识别不可能 100% 准确，强行纠错反而引入错误。识别结果保留模型置信度，低于阈值的算式标记 `low_confidence: true`，供后续校验环节重点复核。
- **常见降级**：
  - 分数识别失败 → 转纯文本 `3/4`；
  - 根号识别失败 → 转纯文本 `sqrt(2)`；
  - 上下标识别失败 → 转纯文本 `x^2`。

### 5.2 渲染

- **前端引擎**：KaTeX（比 MathJax 轻量、首次渲染快 3-5 倍，体积约 280KB）。
- **加载策略**：KaTeX 库与字体子集化打包进前端 bundle，CDN 失效时本地兜底。
- **渲染时机**：题目 JSON 返回后立即同步渲染，避免页面闪烁。

### 5.3 三级降级（绝不崩溃）

```
LaTeX 渲染成功 → 展示渲染后公式
    ↓ KaTeX 抛错
纯文本兜底 → 展示原始 LaTeX 源码（如 "y = ax^2 + bx + c"）
    ↓ 纯文本也无意义（乱码）
原题截图兜底 → 展示识别时保留的原图裁剪片段
```

每一级降级在前端 console 打日志，便于后续统计识别失败率、迭代识别模型。

## 6. Prompt 工程规范

### 6.1 JSON 输出强约束

- **system prompt 锁定 schema**：在 system 段直接给出目标 JSON 的字段名、类型、枚举值，并要求"只输出 JSON，不要任何前后缀文字、不要 markdown 代码块标记"。
- **示例驱动**：在 system 段附 1-2 个完整正确示例，比纯文字描述更稳。
- **解析容错**：即便如此，模型仍可能输出 ```json ... ``` 包裹，解析时先 strip 代码块标记再 `json.loads`。

### 6.2 二维错因标签

错因必须从两个维度同时打标，缺一不可：

- **维度一：知识点**（来自知识图谱）——错在哪一节；
- **维度二：错因类型**（来自图谱 `error_types` 枚举）——错在哪个环节。

`error_types` 枚举（示例）：

```json
[
  {"code": "E-CONCEPT",   "label": "概念混淆"},
  {"code": "E-COMPUTE",   "label": "计算失误"},
  {"code": "E-PROCEDURE", "label": "步骤遗漏"},
  {"code": "E-READ",      "label": "审题错误"},
  {"code": "E-TRANSFER",  "label": "知识迁移失败"},
  {"code": "E-FORMULA",   "label": "公式记错"},
  {"code": "E-LOGIC",     "label": "逻辑推理断裂"},
  {"code": "E-OTHER",     "label": "其他（需备注）"}
]
```

诊断输出形如：

```json
{
  "knowledge_point_id": "KP-FUNC-007",
  "error_type": "E-CONCEPT",
  "error_type_note": "",
  "root_cause_kp": "KP-FUNC-003",
  "tutor_script": "...",
  "makeup_path": ["KP-FUNC-003", "KP-FUNC-007"]
}
```

### 6.3 防越狱 / 防跑题

- **system 段硬声明**："你是初中数学教研助手，只回答与初中数学教学相关的问题。任何与数学教学无关的请求，一律回复'我只处理初中数学教学问题'。"
- **用户输入消毒**：上传的错题文本在送入 LLM 前，做关键词黑名单过滤（政治、暴力、成人等），命中即拒绝并提示用户重新上传。
- **上下文隔离**：诊断 Prompt 与出卷 Prompt 严格分离，不共用历史对话，避免上下文污染。

## 7. 鲁棒性设计

### 7.1 try-except 包裹所有 LLM 调用

```python
async def safe_llm_call(prompt: str, fallback: dict) -> dict:
    try:
        resp = await qwen_client.chat(prompt, timeout=15)
        return parse_json(resp)
    except (TimeoutError, JSONParseError, APIError) as e:
        log.warning(f"LLM 调用失败: {e}, 降级到 fallback")
        return fallback
```

每一处 LLM 调用都必须有对应的 `fallback` 默认值，不允许"裸调用"。

### 7.2 超时重试

- **重试次数**：3 次；
- **退避策略**：指数退避（1s → 2s → 4s）；
- **重试条件**：仅对超时和 5xx 错误重试，4xx（参数错）立即失败不重试。

```python
@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=1, max=4),
    retry=retry_if_exception_type((TimeoutError, API5xxError)),
)
async def call_with_retry(prompt: str): ...
```

### 7.3 LLM 超时 / 异常时降级

| 故障点 | 降级策略 |
|---|---|
| Qwen3-Omni 识题超时 | 提示用户"识别服务繁忙"，错题暂存原图，待网络恢复后重试 |
| Qwen3-Max 诊断超时 | 跳过精细诊断，按"学生最近 3 次错题的知识点众数"做粗粒度推荐 |
| Qwen3-Max 出卷超时 | 降级到"默认推荐题库"——按薄弱知识点从题库取该考点下命中频次最高的 N 道题，不做 LLM 排版 |
| 向量库检索失败 | 降级到本地索引（题库精简版，按知识点 id 哈希分桶缓存） |

### 7.4 JSON 解析失败时二次提示

第一次解析失败时不直接抛错，把模型原始输出 + schema 提示再送回去，让它自我修正：

```python
async def parse_with_self_correct(raw: str, schema_hint: str) -> dict:
    try:
        return json.loads(strip_code_fence(raw))
    except JSONDecodeError:
        correct_prompt = (
            f"你上一条输出不是合法 JSON，请严格按以下 schema 重新输出，只输出 JSON：\n"
            f"{schema_hint}\n\n你的上一条输出：\n{raw}"
        )
        raw2 = await qwen_client.chat(correct_prompt, timeout=15)
        return json.loads(strip_code_fence(raw2))  # 仍失败则抛错，由外层 try-except 降级
```

二次修正仍失败，则交给外层 `safe_llm_call` 走 fallback 路径。

## 附：方案信念

**在数学教学场景，宁可少推荐一道题，也绝不推荐一道错题。**

RAG 是题目的"出生证"，LLM 是题目的"调度员"——调度员可以选、可以排，但不能凭空造题。这是本方案所有设计的出发点，也是衡量任何扩展是否合规的底线。
